¿Qué hace diferente a Claude? Teoría

Claude (Anthropic) destaca por dos capacidades que otros modelos no integran de forma tan fluida en la interfaz de chat: los Artefactos y los Proyectos. Fue entrenado con énfasis en seguir instrucciones complejas con precisión, razonamiento de larga extensión y generación de código funcional.

🧩
Artefactos

Ventana lateral que renderiza en tiempo real el output generado: páginas web funcionales, código ejecutable, documentos Markdown, visualizaciones SVG, apps React. El resultado no es texto: es el artefacto en sí, interactivo y descargable.

📁
Proyectos

Espacio de trabajo con memoria persistente entre conversaciones. Sube archivos de referencia que Claude recordará en todas las sesiones del proyecto: apuntes, código base, guías de estilo, rúbricas.

💡

Flujo de trabajo: pides el artefacto → Claude lo genera en la ventana lateral → lo editas a través del chat ("añade un botón de reseteo", "cambia los colores a paleta oscura") → lo descargas como index.html y lo subes a GitHub Pages.

Caso práctico — App de ejercicios interactivos para TIC Práctica
  1. Crea cuenta en claude.ai (gratuita). Verifica que los Artefactos están activados: menú lateral → configuración → Artefactos: ON.
  2. Inicia un nuevo chat y envía el Prompt Maestro de abajo. Claude generará el artefacto en la ventana lateral mientras responde.
  3. Observa la previsualización en tiempo real: verás la app aparecer mientras Claude escribe el código. Puedes interactuar con ella directamente.
  4. Refina con instrucciones: "Añade una pregunta sobre bucles FOR", "Cambia el diseño a fondo más oscuro", "Añade un contador de racha correcta".
  5. Descarga el artefacto: botón de descarga → archivo index.html → súbelo a tu repositorio GitHub Pages.
  6. Crea un Proyecto si vas a seguir desarrollando la app en varias sesiones: sube el index.html como archivo de referencia para que Claude mantenga la coherencia del código.
Prompt Maestro — Artefacto HTML educativo Prompt
claude-artefacto-ejercicios.txt
# PROMPT MAESTRO — ARTEFACTO HTML EDUCATIVO CON CLAUDE Actúa como desarrollador frontend experto en educación. Crea un artefacto HTML completo (un único archivo, sin dependencias externas) que sea una aplicación web de ejercicios tipo test sobre [TEMA: ej. "tipos de datos en Python" / "álgebra booleana" / "Git"]. REQUISITOS FUNCIONALES: - Mínimo 8 preguntas de opción múltiple (3-4 opciones cada una). - Feedback inmediato: verde si correcto, rojo si no, con explicación breve de la respuesta correcta. - Barra de progreso (pregunta X de 8). - Pantalla de resultados final con puntuación y botón "Repetir". - Orden de preguntas aleatorio en cada intento. REQUISITOS DE DISEÑO: - Dark mode (fondo #0d1117, acentos en azul #58a6ff). - Tipografía sans-serif limpia. Totalmente responsive. - Sin librerías externas: solo HTML, CSS y JavaScript vanilla. REQUISITOS DE CÓDIGO: - Comentado en español. - Preguntas en un array JSON al inicio del script (fácil de editar). - Al terminar, indica cómo añadir nuevas preguntas al array.
Prompt Maestro — Configuración de Proyecto en Claude Prompt
claude-proyecto-instrucciones.txt
# INSTRUCCIONES PERSONALIZADAS DEL PROYECTO CLAUDE # Pega este texto en "Instrucciones del proyecto" al crearlo Eres mi asistente técnico para el proyecto: [NOMBRE DEL PROYECTO]. Contexto: [Describe en 2-3 líneas de qué va el proyecto. Ej: "Estoy desarrollando una app web de gestión de tareas en HTML/CSS/JS para el proyecto final de 2º de Bachillerato TIC."] Archivos adjuntos de referencia: - [index.html]: código base actual de la aplicación. - [guia-estilo.md]: paleta de colores y tipografías a usar. - [rubrica.pdf]: criterios de evaluación del proyecto. Normas de trabajo: - Mantén el estilo de código de los archivos adjuntos. - Comenta el código en español. - Si propones un cambio que rompe funcionalidad existente, avísame antes de implementarlo. - Genera siempre código como artefacto HTML completo y funcional, no como fragmento aislado. Modo de respuesta: Respuestas técnicas y directas. Sin introducciones innecesarias. Cuando detectes un error en mi código, explícame el porqué antes de dar la solución.
