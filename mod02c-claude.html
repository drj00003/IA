① Artefactos Teoría

Un Artefacto es un output que Claude genera y renderiza en tiempo real en la ventana lateral del chat. No es texto plano: es el elemento en sí, funcional e interactivo. Puedes hablar con Claude mientras el artefacto se construye delante de ti y pedirle modificaciones en lenguaje natural.

TipoQué generaCaso de uso TIC
HTML/CSS/JSPáginas web completas, apps interactivasProyectos web, juegos educativos, portfolios
ReactComponentes con estado dinámicoSimuladores, calculadoras, formularios
SVGGráficos vectoriales, diagramasDiagramas de flujo, mapas conceptuales
MarkdownDocumentos estructurados con formatoMemorias, README, guiones de presentación
CódigoPython, JS, SQL con sintaxis resaltadaScripts, consultas, funciones reutilizables
💡

Flujo de trabajo: pides el artefacto → Claude lo genera en la ventana lateral → lo editas con instrucciones en el chat → lo descargas como index.html → lo subes a GitHub Pages.

Markdown → Presentaciones con conectores Práctica

Markdown es un lenguaje de marcado ligero que usa símbolos de texto (#, **, -, ---) para dar formato a documentos sin necesidad de un editor visual. Es el estándar en GitHub, Notion, Obsidian y decenas de herramientas educativas.

Su potencia en presentaciones reside en que varias plataformas aceptan Markdown como fuente para generar slides de forma automática. El flujo es: Claude genera el Markdown → tú lo pegas en el conector → el conector produce la presentación visual.

ConectorCómo usa el MarkdownResultado
Gamma Pega el Markdown en "Importar texto" → genera slides automáticamente Presentación con diseño IA, imágenes y animaciones
Marp Archivo .md con separadores --- entre slides PDF o HTML con diseño limpio y controlable por CSS
Reveal.js Markdown embebido en HTML con separadores de diapositiva Presentación web interactiva con transiciones
Slides.com Importación directa de Markdown por bloques Deck online colaborativo y embebible
ℹ️

¿Por qué Markdown + Claude es más potente que usar Gamma directamente? Porque Claude puede generar el contenido con mucho más control sobre el texto, la estructura pedagógica y el nivel de detalle. Gamma es excelente para el diseño visual; Claude es superior para el contenido técnico. Combinados son imbatibles.

Ejemplo de Markdown para presentación (compatible con Marp y Gamma):

presentacion-markdown.md

El prompt es la interfaz humano-máquina 2. Más contexto = mejores respuestas 3. Documenta tus mejores prompts

② Proyectos — Memoria persistente entre sesiones Teoría

Un Proyecto en Claude es un espacio de trabajo donde puedes subir archivos de referencia e instrucciones que Claude recordará en todas las conversaciones del proyecto, aunque no recuerde los chats anteriores. Es el equivalente a tener un asistente con su propio expediente de tu proyecto siempre abierto.

  1. Sube archivos de referencia que Claude leerá en todas las sesiones: apuntes, código base, rúbricas, guías de estilo, documentos de requisitos.
  2. Escribe instrucciones personalizadas que definen el rol y comportamiento de Claude para ese proyecto específico (equivalente al system prompt de las Gemas).
  3. Trabaja en conversaciones separadas dentro del proyecto: Claude mantiene el contexto de los archivos aunque cada chat sea nuevo.
ℹ️

Los Proyectos son ideales para trabajar durante semanas en un mismo trabajo: una investigación larga, el desarrollo iterativo de una app web, o la preparación sistemática de una asignatura.

claude-proyecto-instrucciones.txt
# INSTRUCCIONES PERSONALIZADAS DEL PROYECTO CLAUDE Eres mi asistente técnico para: [NOMBRE DEL PROYECTO]. Contexto: [Ej: "Estoy desarrollando una app web de gestión de tareas en HTML/CSS/JS para el proyecto final de 2º Bachillerato TIC."] Archivos adjuntos: - [index.html]: código base actual. - [guia-estilo.md]: paleta de colores y tipografías. - [rubrica.pdf]: criterios de evaluación del proyecto. Normas: - Mantén el estilo de código de los archivos adjuntos. - Comenta el código en español. - Avísame antes de implementar cambios que rompan funcionalidad. - Genera código siempre como artefacto HTML completo, no fragmentos. - Respuestas técnicas y directas, sin introducciones innecesarias.
③ Skills — Claude conectado al mundo real Teoría

Las Skills (también llamadas integraciones o conectores MCP) son extensiones que permiten a Claude interactuar con servicios externos en tiempo real: buscar en la web, leer y escribir en Google Drive, ejecutar código, consultar bases de datos o conectarse a APIs de terceros.

A diferencia de los artefactos, que son outputs estáticos, las Skills dan a Claude capacidad de acción: puede leer un documento de Drive, modificarlo y guardarlo, o buscar información actualizada en Internet que va más allá de su fecha de corte de conocimiento.

⚠️

Las Skills con acceso a servicios externos requieren conceder permisos explícitos. Revisa siempre qué permisos solicita cada Skill antes de instalarla. Usa solo Skills de fuentes de confianza.

Dónde encontrar Skills (repositorios externos) Recursos

Las Skills de Claude se implementan mediante el protocolo MCP (Model Context Protocol), un estándar abierto creado por Anthropic que cualquier desarrollador puede implementar. Esto significa que hay un ecosistema creciente de conectores creados por la comunidad.

🐙
MCP Servers (oficial)
Repositorio oficial de Anthropic con servidores MCP mantenidos por el equipo. Incluye Google Drive, GitHub, Slack, Postgres, Brave Search y más. Punto de partida recomendado.
📦
Glama MCP Registry
Directorio de la comunidad con más de 1.000 servidores MCP categorizados por función: productividad, datos, desarrollo, educación, etc.
🔭
mcp.so
Buscador especializado en servidores MCP. Permite filtrar por categoría, popularidad y compatibilidad con Claude Desktop o claude.ai.
⭐
Awesome MCP Servers
Lista curada en GitHub con los servidores MCP más destacados, organizados por categoría. Actualizada por la comunidad semanalmente.
💡

Para empezar sin complicaciones: en claude.ai ve a Configuración → Integraciones para activar las Skills disponibles directamente desde la interfaz web (Google Drive, Docs, búsqueda web). No necesitas instalar nada. Las Skills MCP avanzadas requieren Claude Desktop y algo más de configuración técnica.

Práctica — Flujo con Skills en el aula Práctica
  1. Activa la Skill de Google Drive en Configuración → Integraciones. Autoriza el acceso a tu Google Drive educativo.
  2. Crea un Doc de apuntes en Drive con el contenido de un tema de clase. Copia el enlace del Doc.
  3. En un nuevo chat de Claude, pega el enlace y escribe: "Lee este documento y genera un esquema de estudio en tres niveles de profundidad".
  4. Claude leerá el Doc directamente y generará el esquema como artefacto Markdown. No has copiado ni pegado nada.
  5. Pide que lo guarde: "Crea un nuevo Google Doc con el título 'Esquema Tema 5' y guarda el esquema en mi Drive".
  6. Combina con Web Search: "Busca las 3 noticias más recientes sobre [tema del apunte] y añádelas como sección 'Actualidad' al final del Doc".
claude-skills-prompts.txt
# PROMPTS MAESTROS CON SKILLS ACTIVADAS ── Con Google Drive ── Lee el documento de Drive en este enlace: [URL] Resume el contenido en un esquema Markdown de 3 niveles. Luego crea un nuevo Doc en mi Drive llamado "Resumen [TEMA]" y guarda el esquema en él. ── Con Web Search + Drive ── Busca las últimas noticias sobre [TEMA] de los últimos 30 días. Selecciona las 3 más relevantes para un alumno de bachillerato. Añade un apartado "Contexto actual" al Doc de Drive: [URL] con un párrafo por cada noticia (fuente + resumen de 2 líneas). ── Con ejecución de código ── Tengo este código Python que falla: [pega tu código aquí] Ejecútalo, muéstrame el error real y corrígelo. Explícame en 2 líneas qué estaba mal. ── Con GitHub ── Lee el repositorio: [URL del repo] Analiza el archivo README.md y el código principal. Dime: ¿qué hace bien? ¿qué mejorarías? ¿hay errores evidentes? Crea un issue en el repo con el título "Code Review" y el resumen de tus sugerencias.
🧠 Quiz — Claude: Artefactos, Proyectos y Skills Test

1. ¿Cuál es la diferencia entre un Artefacto y una Skill?

Los artefactos son de pago y las Skills son gratuitas
Los artefactos son outputs generados (webs, código); las Skills conectan Claude con servicios externos para leer y actuar
Son lo mismo, solo cambia el nombre según la versión de Claude

2. Quieres que Claude lea tus apuntes de Drive y genere un resumen sin copiar y pegar. ¿Qué necesitas activar?

La Skill de Google Drive en Configuración → Integraciones
Un artefacto Markdown con el contenido de los apuntes
